### Breaking Changes

<Callout type="warn" title="MessageBox namespace changed">
`MessageBox` moved from `JEngine.Core.Misc` to `JEngine.UI` and is now in the separate `jengine.ui` package. See the [Migration Guide](/en/docs/v1.1/migration) for details.
</Callout>

---

JEngine's design philosophy has always been to allow users (usually those without hot update experience) to download the framework and immediately start developing hot-updatable games. The v1.0+ series is built on HybridCLR for native C# performance.

## Package Architecture

v1.1 organizes JEngine into three installable packages:

| Package | ID | Purpose |
|---------|-----|---------|
| **Core** | `com.jasonxudeveloper.jengine.core` | Hot update runtime, bootstrap, resource management |
| **UI** | `com.jasonxudeveloper.jengine.ui` | MessageBox, editor UI components, theming |
| **Util** | `com.jasonxudeveloper.jengine.util` | JAction, JObjectPool |

## Key Features

<Tabs items={['Lightweight Core', 'Editor Tools', 'Runtime Configuration']}>
<Tab value="Lightweight Core">
Simply download the framework to develop hot-updatable games. Core functionality:

- Package hot update resources
- Download hot update resources
- Enter hot update scenes
- Delete local package cache
</Tab>

<Tab value="Editor Tools">
One-click operations:

- Compile hot update code
- Generate [AOT generics](https://www.hybridclr.cn/docs/basic/aotgeneric)
- Obfuscate code with [Obfuz](https://www.obfuz.com/docs/beginner/configuration)
- Encrypt resources (XOR, AES, ChaCha20)
- Generate packages with [subpackaging](https://www.yooasset.com/docs/guide-editor/AssetBundleCollector)
- Auto shader collection
</Tab>

<Tab value="Runtime Configuration">
Configure through runtime tools:

| Setting | Options |
|---------|---------|
| **Run modes** | Editor, Regular (iOS/Android/Windows/MacOS/WebGL), Standalone, Mini-games |
| **Resources** | Download URL, first package, encryption method |
| **Entry** | Hot update entry point, initial scene |
| **Security** | Obfuscation decryption key |

<Callout type="info">
**Standalone** mode loads resources from StreamingAssets without a server - ideal for offline/single-player games that may need hot updates later.
</Callout>
</Tab>
</Tabs>
