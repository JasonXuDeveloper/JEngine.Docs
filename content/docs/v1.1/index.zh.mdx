<Callout type="warn" title="MessageBox 命名空间变更">
`MessageBox` 从 `JEngine.Core.Misc` 迁移至 `JEngine.UI`，现位于独立的 `jengine.ui` 包中。详见[迁移指南](/zh/docs/v1.1/migration)。
</Callout>

---

JEngine 的设计思路一直是让（通常是无热更新基础的）用户可以下载本框架后就能做支持热更新的游戏。v1.0+ 系列基于 HybridCLR 实现原生 C# 性能。

## 包体系架构

v1.1 将 JEngine 组织为三个可安装的包：

| 包 | ID | 用途 |
|---|-----|------|
| **Core** | `com.jasonxudeveloper.jengine.core` | 热更新运行时、Bootstrap、资源管理 |
| **UI** | `com.jasonxudeveloper.jengine.ui` | MessageBox、编辑器 UI 组件、主题系统 |
| **Util** | `com.jasonxudeveloper.jengine.util` | JAction、JObjectPool |

## 核心特性

<Tabs items={['轻量核心', '编辑器工具', '运行时配置']}>
<Tab value="轻量核心">
仅需下载框架即可开发可热更的游戏。核心功能：

- 打包热更资源
- 下载热更资源
- 进入热更场景
- 删除本地分包缓存
</Tab>

<Tab value="编辑器工具">
一键操作：

- 编译热更代码
- 生成 [AOT 泛型](https://www.hybridclr.cn/docs/basic/aotgeneric)
- 混淆代码（[Obfuz](https://www.obfuz.com/docs/beginner/configuration)）
- 加密资源（XOR、AES、ChaCha20）
- 生成[分包](https://www.yooasset.com/docs/guide-editor/AssetBundleCollector)
- 自动收集 Shader
</Tab>

<Tab value="运行时配置">
通过运行时工具配置：

| 设置 | 选项 |
|------|------|
| **运行模式** | 编辑器、常规（iOS/Android/Windows/MacOS/WebGL）、单机、小游戏 |
| **资源设置** | 下载地址、首包资源、加密方式 |
| **入口配置** | 热更程序入口、初始场景 |
| **安全设置** | 混淆解密密钥 |

<Callout type="info">
**单机模式**从 StreamingAssets 加载资源，无需服务器 - 适用于单机游戏，且未来可随时启用热更新。
</Callout>
</Tab>
</Tabs>
